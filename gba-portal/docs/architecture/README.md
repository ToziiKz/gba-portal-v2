# Architecture Overview

Ce dossier centralise la vue d'ensemble du projet `gba-portal`.

## Documents

- `dashboard-modules.md` : cartographie des modules dashboard et responsabilités.
- `auth-roles-permissions.md` : flux d'authentification et contrôle d'accès (roles/scope).
- `invitation-activation-flow.md` : cycle complet invitation → activation coach.

## Objectif

- Accélérer l'onboarding.
- Réduire les erreurs de compréhension entre backend, UI et RLS.
- Garder une source de vérité claire et courte.
